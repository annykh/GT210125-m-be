Техническое описание базы данных:

База данных "university" содержит 6 таблиц:

1. students (Студенты):
Поля:
  id (PRIMARY KEY, автоинкремент) — идентификатор студента.  
  name (VARCHAR(128), NOT NULL) — имя студента.
  age (INTEGER) — возраст студента.
Содержит информацию о студентах, которые обучаются в университете.

2. teachers (Преподаватели):
Поля:
  id (PRIMARY KEY, автоинкремент) — идентификатор преподавателя.
  name (VARCHAR(128), NOT NULL) — имя преподавателя.
  age (INTEGER) — возраст преподавателя.
Содержит информацию о преподавателях университета.

3. competencies (Компетенции):
Поля:
  id (PRIMARY KEY, автоинкремент) — идентификатор компетенции.
  title (VARCHAR(128), NOT NULL) — название компетенции.
Содержит список профессиональных компетенций, которыми могут обладать преподаватели.

4. teachers2competencies (Компетенции преподавателей):
Поля:
  id (PRIMARY KEY, автоинкремент) — идентификатор записи.
  teacher_id (INTEGER) — идентификатор преподавателя, связанный с таблицей teachers.
  competencies_id (INTEGER) — идентификатор компетенции, связанный с таблицей competencies.
Описывает связь преподавателей с их профессиональными компетенциями.

5. courses (Курсы):
Поля:
  id (PRIMARY KEY, автоинкремент) — идентификатор курса.
  teacher_id (INTEGER) — идентификатор преподавателя, ведущего курс, связанный с таблицей teachers.
  title (VARCHAR(128), NOT NULL) — название курса.
  headman_id (INTEGER) — идентификатор заведующего кафедрой, связанный с таблицей teachers.
Содержит информацию о курсах, которые преподаются в университете, включая преподавателя, ведущего курс, и заведующего кафедрой.

6. students2courses (Студенты и курсы):
Поля:
  id (PRIMARY KEY, автоинкремент) — идентификатор записи.
  student_id (INTEGER) — идентификатор студента, связанный с таблицей students.
  course_id (INTEGER) — идентификатор курса, связанный с таблицей courses.
Описывает связь между студентами и курсами, на которых они обучаются.

Связи между таблицами:
-> teachers2competencies → связь с teachers и competencies:
  Поле teacher_id в таблице teachers2competencies связано с полем id в таблице teachers. Это означает, что каждая запись в таблице teachers2competencies ссылается на преподавателя.
  Поле competencies_id в таблице teachers2competencies связано с полем id в таблице competencies. Это указывает, какими компетенциями обладает преподаватель.

-> courses → связь с teachers:
  Поле teacher_id в таблице courses связано с полем id в таблице teachers. Это указывает на преподавателя, который ведёт данный курс.
  Поле headman_id в таблице courses также связано с полем id в таблице teachers. Это указывает на преподавателя, который является заведующим кафедрой для данного курса.

-> students2courses → связь с students и courses:
  Поле student_id в таблице students2courses связано с полем id в таблице students. Это указывает на студента, записанного на курс.
  Поле course_id в таблице students2courses связано с полем id в таблице courses. Это указывает на курс, на который записан студент.

Пример полной структуры связей:

Таблица teachers2competencies связывает:
  teacher_id → teachers.id
  competencies_id → competencies.id

Таблица courses связывает:
  teacher_id → teachers.id (преподаватель курса)
  headman_id → teachers.id (заведующий кафедрой)

Таблица students2courses связывает:
  student_id → students.id
  course_id → courses.id
